Verson control system -> It is a software tool that trake and manage changes to file over time , most commonly used for software devolpment . 
Types of Verson control system -> localized verson control system , distributed verson control system , centralized verson control system 
localized VCS ->It is manages and stores file changes on a single computer, saving history locally. It is best for individual projects since it lacks collaboration features and is vulnerable to data loss if the local machine fails. These systems work by storing changes as "patches" in a local database, allowing a user to revert to previous versions of a file.
distributed VCS ->It is a type of version control in which every developer has a complete local copy of the entire repository, including its full history. This allows developers to work offline, perform operations like committing, branching, and merging faster, and reduces the risk of data loss since there are multiple full backups. Git and Mercurial are popular examples of DVCS.  
centralized VCS->A centralized version control system (CVCS) uses a single central server to store all project files and their version history. Developers check out a copy of the files from this central repository, make changes locally, and then commit their changes back to the server.This model is often simpler to manage and provides a single source of truth, but it creates a single point of failure and requires a constant internet connection for most operations.
